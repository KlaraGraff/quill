# 阅读器排版、选区交互与凭据 P1 全量测试方案

>
> 更新日期：2026-07-15
>
> 本文只定义人工测试步骤和通过标准。Codex 不执行本方案中的真机、视觉和跨版本人工测试；自动化与静态检查由实施过程另行记录。

## 1. 发布门

所有 P0 用例必须通过，且不得存在凭据丢失、重复系统授权弹窗、错误选区、翻页丢页、章节空白页或旧书无法打开的问题。测试时记录提交 SHA、系统版本、窗口尺寸、书籍格式、阅读模式、页面布局和主题。

## 2. 环境与资料

- Apple Silicon MacBook（Force Touch 触控板）和外接鼠标。
- Windows 11 鼠标和触控板环境。
- 1280x720、1440x900、窄竖屏窗口和横向全屏窗口。
- 原始、阅读纸、灰色、深色四种阅读主题，并与应用浅色/深色主题交叉组合。
- EPUB、TXT、Markdown/HTML 转换书、有文字层 PDF、扫描 PDF。
- 包含英文缩写、所有格、连字符词、中文、混合语言、标点、NFC/NFD 组合字符、RTL 文本和跨行内 DOM 节点文本的样本。
- 保留旧 `encrypted_secrets` 的 v1.4.1 数据副本，以及全新空数据目录。

## 3. P0：单击与指针选区

### SEL-01 单击单词

1. 分别单击 `toward`、`don't`、`teacher's`、`well-known` 和中文词语。
2. 观察点击后到菜单出现前的系统选区。
3. 单击空格、逗号、句号、链接、按钮和已有标记。

预期：完整单词先成为清晰可见的系统选区，随后菜单在选区右侧或上下方避让显示；空白、纯标点和交互控件不建立选区、不打开菜单、不请求 AI。

### SEL-02 拖选边界补全

1. 只拖选 `toward` 中的 `tow`。
2. 从 `toward` 中部拖到 `goal` 中部。
3. 跨行、跨段落和跨 DOM 文本节点拖选。
4. 反向拖选同样的范围。
5. 拖选纯空格或纯标点。
6. 从阅读容器或 Foliate iframe 内开始拖选，移出容器/iframe 后松开，再建立新选区。
7. 对被 `em`、`span` 或标记节点拆开的 `don't`、`teacher's`、`well-known` 重复上述操作。

预期：拖动过程中选区不跳动且菜单不出现；松手后分别补全为 `toward` 和 `toward your goal`；只扩展首尾连续词元，中间文本保持原样；纯空白/标点不扩选且不打开菜单；菜单位置按补全后的最终选区重新计算。移出阅读区域松手也必须可靠结束指针状态，后续键盘选择和拖选仍能打开菜单；长章节松手不得出现与章节总长度成比例的停顿。

### SEL-03 不受影响的输入方式

1. 使用 Shift+方向键建立键盘选区。
2. 对已有选区右键。
3. 双击快速查词，并关闭该选项后重试。
4. 在 macOS 进行 Force Click。

预期：键盘选区和右键不被自动扩词；双击只触发一次原有快速查词；关闭后不快速查词；Force Click 只打开系统词典，不与 Quill 菜单双开。

### SEL-04 主题和遮挡

在四种阅读主题、EPUB/TXT/PDF 文字层及单页/双页中重复 SEL-01/02。

预期：系统选区与手动标记、自动查词标记有明显区别；文字仍可读；菜单不覆盖最终选区，也不会越出窗口边界。

## 4. P0：章节分页与阅读排版

### PAGE-01 文本书章节新页

1. 打开包含 `PART TWO 10` 的测试书，选择左右翻页。
2. 分别使用单页和横屏双页导航到该标题前一页。
3. 切换为上下滚动再定位同一标题。

预期：分页模式按解析器的语义等级分页，而不是按目录嵌套深度猜测。`VOLUME / BOOK / PART / CHAPTER / ACT / SCENE` 从新列或新页开始，`SECTION` 等小节不强制分页；标题前不残留上一章正文；书籍首个标题不产生空白页；滚动模式保持连续。旧缓存自动升级并重新准备，现有阅读位置仍可恢复。

### PAGE-02 EPUB 章节新页

用以下 EPUB 结构重复 PAGE-01：`body > h1`、`section > h1`、`section > header > h1`、TOC 目标落在标题或章节容器，以及 h2–h6 普通小标题。额外准备“多个 Part 位于 TOC 顶层，每个 Part 内嵌 Chapter 与 Section”、无 TOC 但有上述 h1 结构、以及 TOC 只覆盖同一文档部分章节的样本。

预期：TOC 或章节容器能识别的章节标题从新列开始；Part 内的各 Chapter 仍分页，Section 不被提升为章节；无 TOC 或部分 TOC 时只用受限 h1 结构补全；spine 首标题不制造空页；仅凭通用 h2/h3 标签的普通小标题不被误强制分页。

### PAGE-03 双页回落与宽度

1. 横屏选择双页，然后逐步把窗口收窄为竖屏。
2. 再恢复横屏并调整页边距。

预期：横屏为双页，竖屏稳定回落单页并显示说明；正文最大宽度受当前阅读容器约束而非固定常量；页边距在阅读弹层与全局设置间双向同步。

## 5. P1：翻页输入与动画

### TURN-01 默认和自定义按键

验证方向键、PageUp、PageDown、Space、Shift+Space、自定义键和鼠标侧键；在输入框、AI 面板和设置弹层中重复。

预期：所有默认键方向正确；自定义绑定持久化；可编辑控件和弹层内不误翻页。

### TURN-02 四种动画

在 EPUB、TXT 和 PDF 分页模式下分别选择平滑滑动、渐隐、覆盖、无动画，并在单页/双页、前进/后退、快速连按时重复。

预期：动画方向正确且不丢页；PDF 的平滑滑动不得退化为无动画；EPUB/TXT 不得同时叠加两套滑动；渐隐与覆盖在支持 View Transition 的系统使用目标效果，不支持时 cover 与 slide 走同一降级路径；快速连按按顺序执行且不丢页；滚动模式不套用分页动画。

### TURN-03 减少动态效果

打开 macOS/Windows 的“减少动态效果”后重复 TURN-02。

预期：所有非必要动画关闭，Foliate 不保留原生 `animated`，文本阅读器不使用 smooth scroll，翻页仍立即完成。无 View Transitions 环境下，cover 在 EPUB、PDF、TXT 都按设计降级为 slide，不得直接无动画切页。

## 6. P1：设置 UI

### UI-01 控件宽度与 Toggle

在中英文界面检查通用、外观、阅读偏好、阅读辅助和 AI 服务设置；操作所有开关的开/关/禁用状态。

预期：同类行控件右缘和宽度一致；所有 Toggle 滑块始终完整位于轨道内；Tab 焦点、`role=switch` 和可访问名称正确。

### UI-02 动态预览高度

仅在“卡片设计”和“操作菜单”打开右侧预览，在高窗口、矮窗口和窄窗口切换密度、卡片宽度、菜单类型及长内容。

预期：离开两页时预览收回；高窗口充分使用可用高度；矮窗口只在必要区域滚动；菜单工具条不与卡片重叠；设置主面板宽度不跳变。覆盖式预览出现后，Tab 不得进入被遮住的设置控件，焦点进入预览，Escape 先收回预览而不是关闭整个设置窗口，关闭后焦点回到原触发项。

### UI-03 Portal 主题

组合“应用深色+阅读纸”和“应用浅色+阅读深色”，打开阅读设置中的字体、动画等下拉菜单。

预期：Portal 菜单继承当前阅读主题，而不是回落到 `body` 的应用主题；菜单不被滚动容器裁剪。

## 7. P0：本地凭据 P1 与迁移

### CRED-01 全新安装

1. 使用空数据目录启动，新增多套服务和多个 Key。
2. 重启应用，执行模型列表、连接测试、查词、学习卡片和 AI 对话。

预期：所有日常操作只读本地 `secrets.db`，不出现钥匙串授权框；前端只得到掩码；日志、错误和普通 settings 中无明文 Key。

### CRED-02 v1.4.1 一次性迁移

1. 使用含旧 `encrypted_secrets` 的副本升级。
2. 在迁移提示出现前确认系统不自动请求钥匙串。
3. 主动点击迁移，分别验证允许、拒绝和取消。
4. 允许后重启并执行 CRED-01 第 2 步。
5. 构造“v1.4 主密钥缺失 + 仍可读取的旧逐项 Keychain 凭据”混合状态，以及“一条可解密密文 + 一条损坏密文”状态后迁移。

预期：只有用户明确确认迁移后才请求旧钥匙串；允许后所有可读取凭据写入本地表并只清理对应旧密文；主密钥缺失或单条密文损坏不会阻断其他独立可恢复项，失败行保持原样、界面继续显示待迁移状态并给出明确错误；拒绝/取消不循环弹窗且旧数据仍可再次迁移；全部成功后日常使用永久退出钥匙串热路径。

### CRED-03 文件保护与删除

检查 `secrets.db` 权限、`secure_delete`，替换和删除 Key 后搜索数据库可见记录，并验证应用 API 输出。

预期：数据库文件权限为 0600，使用不会把旧明文长期留在 WAL 的日志模式，并启用 secure delete；产品明确接受本地同用户进程可读的明文权衡，但不宣称可抵御磁盘取证；删除后业务查询不可恢复该 Key；Key 不参与当前 iCloud 事件、快照或日志同步。

### CRED-04 跨数据库补偿失败

分别对新增、替换、删除凭据注入主数据库写失败和 `secrets.db` 补偿失败，再重启应用核对元数据和本地秘密引用。

预期：调用方收到同时包含主操作与补偿结果的明确错误，不得静默成功；失败后的不一致状态有日志和可定位错误，不会把补偿失败伪装为普通业务失败，也不会在无记录的情况下覆盖已有凭据旧值。

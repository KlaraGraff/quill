# Quill Personal v1.4.1 全量测试方案

> 状态：待执行
>
> 更新日期：2026-07-14
>
> 本文只定义测试范围、步骤和通过标准，不记录任何已通过结论。本轮不执行测试、运行应用、构建或打包。

## 1. 目标与发布门

验证阅读辅助功能在真实阅读、异常、升级和跨设备场景下满足对齐规格，并确认不会破坏既有书库、阅读位置、手动标记、收藏、笔记和同步数据。

所有 P0/P1 用例通过且无未解决的阻断缺陷后，才允许创建 Tag 和发布安装包。macOS 只验证 Apple Silicon，不再生产或验证 Intel 安装包。

## 2. 环境矩阵

| 维度 | 最低覆盖 |
|---|---|
| macOS | Apple Silicon，当前支持的最低系统版本与最新稳定版本 |
| Windows | Windows 11 x64 |
| 安装状态 | 全新安装、从上一可用版本升级、保留现有书库升级 |
| 数据库 | 空库、schema 20、schema 21、schema 22 |
| 显示 | 1280x720、1440x900、窄窗口、系统缩放 125%/150% |
| 网络 | 正常、离线、高延迟、超时、限流、服务端 5xx |
| 同步 | 单机关闭同步、两台 Mac 共用同一 iCloud Drive 文件夹 |

每轮记录应用版本、提交 SHA、系统版本、数据库来源、书籍样本、AI 服务和测试时间。测试数据必须使用专用书库和专用 API Key，不得直接操作唯一的正式书库。

## 3. 测试资料

至少准备：

- 标准 EPUB：英文小说，包含章节、脚注、链接、斜体、长段落和重复单词。
- 复杂 EPUB：出版社 CSS、嵌套元素、跨节点单词、长章节和多语言字符。
- TXT、Markdown、HTML：UTF-8、中文加英文、超长行、空行、章节标题和特殊字符。
- PDF 文本版：Word 导出或原生文字层，单页与多页。
- PDF 扫描版：纯图片页，以及“部分页面有文字层、部分页面为扫描图”的混合文件。
- 一本已有手动标记、收藏、查询记录、笔记和阅读进度的旧书库副本。
- 合法 TTF、OTF、WOFF、WOFF2 字体，以及损坏文件、伪装扩展名和超大文件。
- 两套可用 AI 服务、一个失效 Key、一个限流 Key、一个不可用 URL 和一个不支持模型列表的兼容服务。

## 5. 数据迁移与兼容

### MIG-01 全新数据库

1. 使用空数据目录启动当前版本。
2. 检查迁移版本及 021/022 新表、索引和默认设置。
3. 导入一本书并创建查询、笔记、手动标记和自动标记。

预期：表结构完整；新数据可写入和读取；没有重复迁移或空字段错误。

### MIG-02 旧库升级

1. 分别复制 schema 20 和 schema 21 的真实库。
2. 升级后检查原书籍、阅读进度、收藏、手动标记、旧查询记录和旧 highlight note。
3. 打开旧查询记录，再重新查询同一内容。

预期：旧数据不丢失；旧查询使用兼容卡片；重新查询后写入结构化结果；旧 highlight note 只迁移一次。

### MIG-03 升级失败恢复

1. 使用只读目录、磁盘空间不足副本或人为中断模拟迁移失败。
2. 恢复环境后重新启动。

预期：数据库事务回滚；不会留下“版本已升级但表不完整”的状态；用户收到可操作错误。

## 6. AI 服务配置

### AI-01 服务档案与排序

1. 新建至少三套服务，分别配置名称、Provider、URL、Key 和 Model。
2. 收起卡片后拖动顺序，停用其中一套，再关闭并重开设置。
3. 删除服务并检查系统凭据存储引用。
4. 在长服务名、长模型名、窄窗口下反复折叠、展开卡片并重新打开设置。

预期：折叠卡片完整显示服务摘要；拖动顺序持久化；停用服务不参与路由；删除失败不会造成数据库与凭据分叉；状态圆点、连接测试和模型列表图标始终位于卡片边界内，不溢出也不挤压摘要文字。

### AI-02 模型发现

1. 在未保存的草稿中填写 URL、Provider 和 Key，点击模型列表图标。
2. 分别覆盖 OpenAI-compatible、Anthropic、Ollama 和不提供列表端点的服务。
3. 搜索并选择模型，再保存配置。

预期：发现使用当前草稿而非旧值；模型去重排序；失败时保留手工输入；Key 不出现在 UI、日志或错误文本。

### AI-03 连接测试

1. 在折叠卡片点击无文字测试图标。
2. 覆盖成功、401、429、超时、5xx、无效 Model 和无效请求参数。

预期：发送最小真实推理请求；显示成功/失败、延迟和脱敏原因；模型列表成功不能代替连接测试成功。

### AI-04 Key 与服务故障切换

按以下顺序执行一次查词并记录实际命中项：

| 场景 | 预期 |
|---|---|
| 主 Key 失效，备用 Key 可用 | 同服务切换到备用 Key |
| 同服务全部 Key 不可用，下一服务可用 | 切换到下一完整服务 |
| 429、网络、超时、5xx | 在尚未输出时按策略重试或回退 |
| 400/参数错误 | 不盲目轮换全部 Key 和服务 |
| 已输出部分内容后连接中断 | 不拼接另一模型的回答 |
| 用户取消请求 | 不触发后台故障切换 |

预期：顺序与折叠卡片一致；冷却状态可恢复；查询记录保存实际服务和模型；错误中不含 Provider 原始响应体或 Key。

### AI-05 有效输入自动保存

1. 新建和编辑服务名、Provider、URL、Model 及 Key，分别通过停止输入、移出焦点和直接关闭设置触发保存。
2. 输入空名称、非法 URL、空 Model 和超长字段，再恢复为合法值。
3. 在自动保存进行中快速连续修改同一字段，再关闭并重开设置。
4. 模拟保存失败，然后恢复数据库可写并再次修改。

预期：合法草稿无需额外点击保存；无效值不覆盖最后一份有效配置并显示就地提示；旧的异步保存结果不得覆盖较新输入；失败可见且可重试；Key 仍仅通过安全存储流程更新。

## 7. 阅读触发与格式

对 EPUB、TXT/Markdown/HTML 内部文本阅读器和有文字层 PDF 重复执行：

### INT-01 单击与双击

1. 单击单词中央、词间空白、行末空白、链接、按钮和已有标记。
2. 双击单词，随后关闭“双击快速查词”再重复。
3. 在 macOS 使用 Force Click 调用系统词典。

预期：单击单词只开菜单且不请求 AI；空白和控件不触发；双击只发起一次快速查词；关闭后保留系统选词；Force Click 不被应用覆盖。

### INT-02 短语与句子选区

1. 拖选短语、完整句子、跨行段落和跨 DOM 节点内容。
2. 保持选区约 150ms，改变选区，再右键打开备用菜单。
3. 翻页或滚动后重复。
4. 建立选区后立即切换书籍、关闭阅读页，或切换 PDF 阅读模式触发视图重建。

预期：选区稳定后菜单锚定右上方并做视口避让；改变选区不会弹出旧菜单；右键不产生重复菜单；翻页手势不误触；已销毁视图的延迟回调不会在新书、新页面或新视图中打开旧菜单。

### INT-03 TXT 后台准备

1. 首次导入 TXT、Markdown 和 HTML。
2. 导入后留在书库观察准备状态，再打开书籍。
3. 模拟准备失败并重试。

预期：转换在导入后后台开始；打开书籍不会长期停在 0%；失败有明确原因和重试入口；原始文件保留。

### INT-04 PDF 文本层

1. 在原生文字 PDF 中选择、查词、解释、翻译和手动标记。
2. 在扫描 PDF 点击画布，并等待页面与文字层完成。
3. 在混合 PDF 切换文字页和扫描页。

预期：有文字层页面正常使用阅读辅助；只有确认无文字层的扫描页显示说明；加载中的页面不提前误报；扫描页不提供伪造选区。

## 8. 操作菜单与学习卡片

### CARD-01 菜单配置

1. 分别配置单词、短语、句子菜单的启用项和顺序。
2. 开启“仅翻译”，关闭其他可选项，并重开应用。
3. 检查未标记、当前位置已标记和全书同词已标记状态。

预期：真实菜单与动态预览一致；单词主动作是查词，短语是释义，句子是解读；已标记单词显示“取消当前标记”和“取消全书同词标记”。

### CARD-02 模块、密度与宽度

1. 对三类卡片切换精简/标准/详细、自动/紧凑/宽。
2. 调整模块顺序、显示、默认展开、模块密度、例句数和重点词数。
3. 在窄窗口和长单词、长标题、长 AI 内容下打开卡片。

预期：必要模块不可关闭但可排序；内容位于所属模块；目标宽度符合矩阵并受阅读区约束；请求过程中宽度不跳动；超过 75vh 时仅内容区滚动。

### CARD-03 卡片操作与笔记

1. 验证底部顺序为收藏、问 AI、笔记、复制。
2. 新建、编辑和删除本书单词笔记、全局单词笔记、短语笔记和句子笔记。
3. 在相同单词或相同位置再次打开卡片。

预期：笔记回显且范围正确；问 AI 带入引用和位置；复制内容完整；收藏与笔记、查询、标记互不隐式创建。

### CARD-04 渐进流式卡片

1. 使用普通模型和思考模型分别查词、解读短语和解读句子。
2. 记录首个可用模块、后续模块和最终完成的出现时间，覆盖缓慢分块、多字节字符被切分和 JSON 字符串中包含括号的情况。
3. 在仅有部分模块时取消、关闭卡片、重新查询，并模拟最终结构化结果校验失败。

预期：不等待整个响应即按完整模块逐步显示；未完整字符串、对象或数组不提前呈现；流式预览不反复重排已稳定模块；最终以 Rust 校验后的完整结果为准；旧请求不能覆盖新卡片。

### CARD-05 卡片拖动与追问上下文

1. 在生成前、流式生成中和完成后拖动卡片标题栏，再缩放窗口、展开/收起目录与 AI 侧栏。
2. 尝试将卡片拖出阅读区四边，并在卡片内选文、滚动、点击链接和底部操作。
3. 生成完整卡片后点击“问 AI”，分别追问卡片中的当前语境义、例句和语法分析，再关闭并恢复该对话。

预期：仅标题栏启动拖动；卡片始终被夹在当前阅读视口内；拖动不中断流式请求；内容交互不误拖；“问 AI”的首条消息同时包含原文、位置和本次完整卡片分析，恢复对话后仍可继续针对该结果追问。

## 9. 动态预览

### PRE-01 本地预览

1. 进入“阅读辅助”；确认默认设置宽度不跳变。
2. 点击“卡片设计”和“操作菜单”，再切换到“交互”和“正文标记”。
3. 修改语言、CEFR、模块、密度、顺序和菜单项，同时监控网络请求。

预期：只有前两页展开右侧预览；离开即收回；抽屉外框稳定、内部卡片响应配置；本地预览不调用任何 AI/API。

### PRE-02 真实预览

1. 显式点击“生成真实预览”。
2. 在请求中关闭抽屉、切换类型或再次生成。

预期：仅显式操作访问 AI；旧请求被取消且结果不能覆盖新预览；错误可见并保留本地预览。

## 10. 标记图层组合

手动范围标记、当前位置查词标记和全书同词规则必须作为独立数据层验证。

| 初始状态 | 执行“取消当前标记” | 执行“取消全书同词标记” |
|---|---|---|
| 仅手动 | 只删除所选手动范围 | 不显示该操作 |
| 仅当前位置自动 | 禁用该 occurrence，再次查词不自动恢复 | 不显示该操作 |
| 仅全书规则 | 为当前位置写例外，其他同词保留 | 关闭规则并清除同词 occurrence |
| 手动 + 当前位置自动 | 两层都从当前位置消失 | 不显示该操作 |
| 手动 + 全书规则 | 删除当前位置手动层并写规则例外 | 关闭规则，手动层保留 |
| 当前位置自动 + 全书规则 | 禁用 occurrence 并写规则例外 | 两类自动层关闭 |
| 三层同时存在 | 三层均从当前位置消失，其他同词仍由规则控制 | 自动层关闭，手动层保留 |

另验证：

1. 部分选区已标记时第一次操作覆盖整个选区，第二次取消整个选区。
2. 从较大手动范围中取消子范围时保留两侧内容。
3. 关闭后再开启同词规则会建立新基线，旧当前位置例外不会意外复活。
4. “清除本书全部查词标记”清除规则、occurrence 和例外，但保留手动标记、收藏、笔记和查询记录。
5. 手动与自动样式分别验证背景、下划线、粗体、字体、颜色、透明度及“自动跟随手动”。
6. EPUB/PDF 的能力降级不改写正文排版；文本阅读器完整应用字体和粗体。

## 11. 同步与乱序

使用两个独立设备 ID 和同一同步文件夹执行：

1. 笔记、全书规则、当前位置例外和 occurrence 分别新增、修改、删除并双向同步。
2. 人为调整事件到达顺序：例外先于父规则、父规则先于例外、相同时间不同设备。
3. 在父规则尚未到达时执行“清除本书全部查词标记”，再让旧父规则到达。
4. 重复回放同一事件，并从快照恢复后再次回放尾部事件。
5. 一端删除书籍，分别选择删除书籍笔记和保留为 detached；另一端仍有延迟事件。
6. 使用旧 schema v1/v2 事件与新 schema v3 快照组合恢复。
7. 让一台设备的事件时间明显领先本机，先接收该事件，再在本机修改同一实体；覆盖应用启动前已存在和运行中刚到达两种情况。

预期：所有设备最终状态一致；孤立例外在父规则到达前不可见；批量清除能重置孤立例外；重复回放幂等；删除书籍不产生孤立聊天、标记或位置；全局/保留笔记按策略脱离书籍；本地新操作使用晚于已观测远端状态的逻辑时间，不会因设备时钟偏移被旧状态覆盖。

## 12. 自定义字体

1. 分别导入 TTF、OTF、WOFF、WOFF2，验证显示名称来自字体元数据且正文真实换字。
2. 重复导入同一文件，导入损坏、空、超大或伪装格式文件。
3. 将字体设为阅读字体、手动标记字体和自动标记字体，重开应用与新章节。
4. 删除正在使用的字体；同时模拟文件重命名/删除失败和数据库事务失败。
5. 在多个窗口观察字体列表与已挂载 EPUB 章节刷新。

预期：合法字体只存于本机应用数据目录且不进入同步；重复导入不重复建档；非法文件被拒绝；删除成功后阅读字体回到系统、标记字体回到跟随原文；删除失败能恢复 staged 文件和数据库记录，不出现悬空引用。

## 13. CEFR 与考试估算

1. 验证讲解语言顺序：自适应双语、英语（按CEFR）、中文；翻译语言独立生效。
2. 在 A1/A2 选择纯英文，验证一次性非阻断提示；再切换 B1-C2。
3. 对 IELTS、TOEFL iBT、TOEIC L&R、Cambridge、DET、CET-4/6 输入边界值、有效步长、越界值和非法小数。
4. 输入阅读分数、未来日期、多项冲突成绩和两级以上差异。
5. 保存、重开、删除记录并应用估算等级。

预期：输入框显示有效范围；前后端校验一致；记录可保存；阅读分数优先；冲突显示范围并要求确认；CET 始终标为低可信度；手动等级始终可覆盖估算。

## 14. 无障碍、窗口与隐私

1. 只用键盘完成设置导航、菜单、卡片、模型发现、连接测试、笔记和关闭操作。
2. 使用 VoiceOver/Narrator 检查图标按钮名称、菜单角色、焦点顺序、加载和错误状态。
3. 在窄窗口、长文本、200% 文本缩放和高对比度下检查溢出与遮挡。
4. 检查日志、同步文件、数据库导出和前端错误对象。

预期：焦点不会逃出模态框；Esc 和方向键行为一致；关闭后焦点返回触发点；文本不重叠；API Key、Provider 原始错误体和非必要大段正文不会写入日志或同步。

## 15. 阅读排版、翻页与性能

### READ-01 容器约束的动态正文宽度

1. 对 EPUB、TXT、Markdown 和 HTML 在窗口、全屏和窄窗口中分别使用单页与双页。
2. 从无侧栏状态依次展开目录、AI 侧栏，连续拖动侧栏宽度并缩放主窗口。
3. 将页边距从 0% 调到 30%，切换字号、字体、行距和横竖阅读模式。
4. 在开发者工具中记录外部阅读容器、Foliate 渲染器/TXT 文章及单栏的实时宽度。

预期：正文不再受 `900px/1000px` 固定上限限制，可扩大到当前外部阅读容器的可用宽度；该容器宽度同时就是动态最大值，任何正文、单栏或双栏之和都不得越界；出版社自带的较窄排版可保留；宽度只随容器和设置变化，不随加载文案抖动。

### READ-02 翻页模式与双页

1. 在 EPUB、TXT/转换文本和有文字层 PDF 中切换上下滚动与左右翻页。
2. 在左右翻页中切换单页/双页和滑动/无动画，使用触控板、滚轮、页面按钮及目录跳转。
3. 保持双页偏好，将阅读容器从横向拉成纵向/窄窗口，再恢复为横向；分别观察 EPUB、TXT 和 PDF。
4. 在首页、末页、奇数总页数、超长章节和模式切换后恢复阅读位置。

预期：模式切换不丢失当前位置；双页每次精确移动一个 spread；纵向/窄容器只临时回退为单页，不改写用户的双页偏好，恢复足够宽度后自动返回双页；排版、页码范围和 PDF 适宽缩放使用同一实际页数；末页无额外空白 spread；翻页动画不改变页数与位置计算；上下滚动时不触发横向锁页。

### READ-03 键盘与鼠标自定义翻页键

1. 验证默认左/上为上一页、右/下为下一页，并分别录入字母键、组合键、鼠标中键、侧键和右键。
2. 尝试录入单独修饰键、鼠标主键、与另一方向重复的按键，并用 `Esc` 取消录入。
3. 在正文、输入框、AI 对话、菜单、卡片及设置录制界面中按相同键。
4. 将右键设为翻页后发生一次翻页，再恢复为其他按键并验证正常右键菜单。

预期：录入即时显示并持久化；冲突键交换而不丢失；主键保留给选文和普通交互；输入控件不误翻页；右键翻页时只屏蔽紧随该次的菜单，普通右键不受影响。

### READ-04 进度显示与页边距同步

1. 在滚动和翻页模式中逐页/逐段移动，切换章节，并开关整书进度与页码。
2. 检查 EPUB 当前渲染章节页、TXT 物理栏和 PDF 文档页的页码语义。
3. 在阅读界面修改页边距，打开外部“阅读偏好”检查；再从外部修改并观察已打开的阅读器。

预期：左下角默认始终显示当前章节进度；整书进度与页码仅按选项追加；双页显示可见页范围；无总页数时不伪造值；两个页边距入口双向同步，无循环写入和设置跳回。

### READ-05 滚动和位置保存性能

1. 使用长章节 EPUB 和大型 TXT，在无标记、大量手动标记、大量同词规则三种状态下持续快速滚动 60 秒。
2. 打开性能记录，检查每个滚动帧的 DOM 查询/几何读取、React 提交次数、主线程长任务和阅读进度数据库写入频率。
3. 滚动中同时流式生成学习卡片或 AI 对话，然后离开并重开书籍检查最终位置。

预期：滚动处理不再每帧全量扫描正文 DOM；块几何只在内容、宽度或排版变化时重建；进度写入串行且受节流控制；流式 UI 更新不造成持续长任务；恢复位置与离开前一致。

## 16. AI 对话流式输出与思考过程

### CHAT-01 回答与思考分流

1. 分别使用显式返回 reasoning 的模型、仅返回答案的模型和将思考混入普通文本的非标准兼容服务。
2. 观察首个 reasoning delta、首个 answer delta 和完成时间，展开/收起思考区并在输出中滚动离开底部。
3. 输出中取消、切换对话、关闭侧栏，然后重开对话。

预期：服务商明确提供的思考过程与答案分区流式显示；不伪造或从普通答案猜测思考过程；不支持时不显示空思考区；思考与答案均可恢复；取消后无残留 delta；用户离开底部后不被强制拉回。

### CHAT-02 渲染节流与完整上下文

1. 用高频小 token 分块输出长回答，记录 React 渲染、滚动更新和主线程长任务。
2. 从学习卡片“问 AI”进入后连续追问，检查发送给模型的上下文边界和保存后的消息内容。

预期：delta 在每个动画帧合并刷新，不为每个 token 单独提交 UI；自动跟随仅在用户仍靠近底部时生效；卡片原文、位置和完整分析只注入该对话的结构化上下文，不泄露到其他书籍或对话。

## 17. 缺陷记录与退出标准

每个失败项记录：用例 ID、环境、复现步骤、预期、实际、截图/日志、数据目录副本和首次失败提交。修复后只重跑相关用例不足以发布，必须再执行受影响模块及一轮核心回归。

发布退出标准：

- P0/P1 全部通过，无数据丢失、密钥泄露、崩溃、无限加载或错误同步。
- P2 缺陷有明确评估、规避方式和版本安排。
- 升级与回滚资料可恢复。
- Apple Silicon 与 Windows 安装包分别完成签名/安装/启动验证。
- 测试记录由执行者填写，不以本文“计划存在”视为测试通过。

# 阅读器、AI 索引与防剧透人工测试方案

> 状态：待人工执行
>
> 更新日期：2026-07-15
>
> 本文只定义人工测试步骤和通过标准。Codex 未执行本文中的真机手势、视觉、真实服务商或跨设备测试；自动化与静态检查由实施过程单独记录。

## 1. 发布门

P0 用例必须全部通过。不得存在一次触控板手势翻多页、侧栏拖拽丢失阅读位置、引用跳转错误、防剧透越过阅读位置、用户编辑摘要被静默覆盖、全书重答重复保存用户消息或索引配置导致凭据泄露的问题。

每个用例记录提交 SHA、系统版本、窗口尺寸、书籍格式、阅读模式、AI 服务、模型、嵌入维度、结果、日志时间范围和截图或录像。

## 2. 环境与样本

- Apple Silicon MacBook，Force Touch 触控板，当前支持的 macOS 和 WKWebView。
- 外接鼠标，以及一台 Windows 11 鼠标或触控板设备。
- EPUB、TXT、Markdown/HTML 文本书、有文字层 PDF、扫描 PDF、MOBI/FB2/CBZ。
- 短书（低于全文注入阈值）和长书；章节内含唯一检索词及后文章节敏感情节。
- OpenAI 兼容云端对话服务、Ollama 或 LM Studio 本地嵌入服务、至少一套故意不可用的备用服务。
- 两台开启文件夹同步的 Mac，用于验证索引本地重建和摘要/消息替换同步。

## 3. P0：阅读器交互

### READER-01 Force Touch 与词语操作

1. 在 macOS 系统设置中保持“用力点按与触觉反馈”开启。
2. 在 EPUB、TXT 和有文字层 PDF 中对单词执行普通单击、双击和 Force Touch。
3. 对缩写、所有格、连字符词、中文和纯标点重复。

预期：Force Touch 仅走系统既有行为，不与 Quill 菜单双开；普通单击和双击遵循应用当前设置；纯标点和空白不触发查词；系统选区、手动标记和自动词语标记彼此可辨。

### READER-02 WKWebView 触控板翻页

1. EPUB、PDF、TXT 分别切换到分页模式。
2. 执行一次长滑并等待完整惯性尾部结束。
3. 快速连续滑动两次，再执行方向反转和横向滑动。
4. 切换滚动模式并重复；执行 macOS 捏合缩放。

预期：每个连续手势最多翻一页或一个双页组；两次主动滑动翻两次；反转立即生效；滚动模式保留原生滚动；`Ctrl+wheel` 形式的捏合不被当作翻页。

### READER-03 侧栏缩放与阅读位置

1. 在 EPUB、PDF、TXT 的单页和双页模式打开 AI/目录侧栏。
2. 连续拖动侧栏宽度，停在最窄、中间、最宽位置。
3. 拖动期间和结束后翻页、跳目录、关闭侧栏并恢复。
4. 在长 EPUB 章节和较大 PDF 中重复。

预期：阅读内容跟随可用容器宽度重排；TXT 不露出半列且回到原锚点；EPUB 不逐帧卡死或漂移；PDF 面板拖动不触发无意义重排；拖动结束后阅读位置、进度和双页组稳定。

### READER-04 引用跳转

1. 让 AI 回答包含同一来源的多个内联引用和底部来源按钮。
2. 在 EPUB、TXT、PDF 中分别点击内联引用与底部来源。
3. 使用片段在 EPUB 中不存在或被截断的引用测试降级。

预期：所有引用均可点击；TXT 精确到字符范围，PDF 到目标页，EPUB 优先定位并闪烁目标片段，检索失败时只降级到章节开头；页面不闪回空链接。

## 4. P0：真实 AI、嵌入与索引

### INDEX-01 嵌入服务配置

1. 配置 Ollama/LM Studio 的 OpenAI 兼容 embeddings URL 和模型，不填密钥并测试。
2. 配置云端嵌入 URL、模型和密钥并测试。
3. 切换到不同输出维度的模型后再次测试。
4. 重启应用并检查状态；搜索日志和前端调用结果中的明文密钥。

预期：探测显示实际维度和延迟；本地端点允许空密钥；云端密钥只进入秘密存储；维度变化重建向量表，只回填匹配模型和维度的缓存；旧维度不会混入新索引；日志和 webview 响应无明文密钥。

### INDEX-02 增量更新与完整重建

1. 打开一本已完成索引的书，记录块数、向量数、章节摘要和概览。
2. 执行增量更新，再执行完整重建。
3. 中途取消一次真实模型生成并重试。
4. 编辑章节摘要和概览后执行增量更新与普通重新生成。
5. 最后显式确认覆盖用户编辑摘要。

预期：无变化的增量更新不重复工作；缺失向量和过期摘要被补齐；完整重建清理并重建派生索引；取消不留下“完成”假状态；用户编辑摘要在未明确确认前保持不变；确认后才覆盖。

### INDEX-03 摘要模型和故障切换

1. 将摘要模型设为“跟随对话模型故障切换”，让首个服务不可用。
2. 改为指定的启用服务并生成另一本文档摘要。
3. 停用该指定服务并观察错误状态。

预期：跟随模式按已配置顺序故障切换；指定模式只使用选定服务；不可用状态有明确错误，不静默改用未授权服务。

### INDEX-04 索引管理界面

1. 从书籍网格、列表、右键菜单和 AI 面板进入索引管理。
2. 编辑概览和章节摘要，浏览只读块列表，测试窄窗口和长文本。
3. 关闭重开并检查内容；在另一台同步设备检查摘要。

预期：入口行为一致；编辑自动或显式保存符合界面反馈；块文本不可修改；长内容不溢出或遮挡；用户编辑状态持久化并同步，向量和块索引不参与同步而在各设备本地重建。

## 5. P0：防剧透

### SPOILER-01 三种格式 cutoff

1. TXT 阅读到某段中部，询问只出现在后文的唯一词和情节。
2. EPUB 阅读到章节中部，分别询问本章和下一章内容。
3. PDF 阅读到第 N 页，询问第 N 页和第 N+1 页内容。
4. 清空或损坏保存位置后重试。

预期：TXT 不检索 `char_start` 晚于当前位置的块，且只使用完整读完章节的摘要；EPUB 当前整章可用、后续章节不可用；PDF 当前页及之前可用、后页不可用；位置缺失时保守限制在开头；所有引用均位于允许范围。

### SPOILER-02 全文、BM25、向量和邻接路径

1. 分别关闭和开启向量检索，在长书中询问后文章节的唯一词。
2. 使用低于全文阈值的短书重复。
3. 让命中块位于 cutoff 前一块，后邻接块位于 cutoff 后。

预期：BM25、向量、相邻块扩展和短书全文注入都不能越过 cutoff；受保护短书上下文随位置变化，不被错误复用为全书缓存。

### SPOILER-03 全书意图与单次覆盖

1. 依次提问“总结全书前半部分”“结局是什么”“How does this story end?”和普通章节问题。
2. 检查全书意图回答下的提示和普通回答的轻量角标。
3. 点击“结合全书重新回答”，观察流式替换、引用和思考过程。
4. 在生成中取消，再制造一次服务错误；最后成功完成并重启应用。

预期：意图检测只显示选择，不会静默解锁；单次覆盖使用全书且不改变本书开关；重答在原助手消息位置流式替换，不重复用户消息；取消或错误恢复原回答；成功后新回答、来源和元数据持久化，重启后仍一致。

### SPOILER-04 设置优先级

1. 全局默认开启，本书关闭；全局关闭，本书开启；再切换回“跟随全局”语义的未覆盖新书。
2. 删除一本有本书覆盖设置的书。

预期：本书覆盖优先于全局默认；切换立即影响下一次请求；删除书籍同时清理无主覆盖键；历史回答不被回溯改写。

## 6. P1：同步与兼容

### SYNC-01 摘要与重答消息

1. 设备 A 编辑摘要并完成一次全书重答替换。
2. 设备 B 同步事件，再由 A/B 在接近时间内替换同一助手消息。
3. 触发日志压缩，应用快照到空数据库并重放尾部事件。

预期：`chat.message.replace` 仅能更新既有助手消息，不能创建缺失消息或覆盖用户消息；并发结果按 `(updated_at, updated_by_device)` 确定收敛；事件直放与快照应用结果相同。

### SYNC-02 旧版本边界

1. 使用保留的 v1-v5 事件和快照升级到当前版本。
2. 用旧客户端尝试读取含 v6 替换事件的日志副本。

预期：当前版本继续读取旧数据；旧客户端明确拒绝不理解的 v6 envelope，不得静默跳过并推进水位；升级前做好共享目录备份，禁止让旧客户端写回覆盖新状态。

## 7. 结果记录

每个用例记录“通过 / 失败 / 阻塞 / 未执行”。未执行不得记录为通过。失败项必须包含最小复现步骤、相关书籍和服务配置（密钥打码）、预期与实际结果、日志时间范围以及缺陷编号。

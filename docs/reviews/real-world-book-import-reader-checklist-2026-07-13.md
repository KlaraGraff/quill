# Quill 真实使用场景检查清单

> 用途：在新版发布前，系统检查导入、解析、打开、阅读、学习功能、同步与异常恢复链路，重点防止“导入后长时间停在 Preparing book 0%”一类真实使用问题。  
> 范围：macOS Apple Silicon 和 Windows；不再要求 macOS Intel 版本。  
> 执行要求：每项都必须记录操作、实际结果和证据；不允许只记“通过/不通过”。

## 1. 检查记录模板

每个用例执行前先复制下表：

| 字段 | 记录内容 |
| --- | --- |
| 用例编号 | 例如 `IMP-TXT-01` |
| 应用版本 | 版本号、Git tag、commit |
| 系统环境 | 操作系统版本、芯片/架构、内存、可用磁盘空间 |
| 安装来源 | 新安装/覆盖升级，安装包名称 |
| 存储方式 | 本地/iCloud，iCloud 是否已下载到本地 |
| 网络条件 | 正常/断网/高延迟/代理 |
| 测试书籍 | 文件名、格式、大小、编码、章节数、文件哈希 |
| 前置状态 | 首次导入/已存在同名书/有旧阅读进度等 |
| 操作步骤 | 按顺序记录用户操作和等待时间 |
| 实际结果 | 界面状态、进度、耗时、是否可继续操作 |
| 证据 | 截图/录屏、时间戳、日志路径及关键日志、产物路径 |
| 结论 | 通过/失败/阻断/未执行 |
| 问题级别 | P0/P1/P2/P3 |
| 问题编号 | Issue/PR 链接 |

### 通用通过标准

- [ ] 每次操作都有明确的进行中、成功或失败状态，不出现无期限等待。
- [ ] 成功后的书籍可立即打开，首屏有正文，不长时间停留在 `Preparing book...` 或 `0%`。
- [ ] 失败后向用户说明原因，并提供重试、重新选择文件或返回书架的可行路径。
- [ ] 一次失败不会使后续导入、打开或退出应用持续失效。
- [ ] 界面显示的格式、书名、作者、章节数和阅读进度与实际渲染内容一致。

## 2. 安装、启动与升级

### `APP-01` macOS Apple Silicon 全新安装

- [ ] 操作：在未安装过 Quill 的用户环境中安装当前发布包，完成 macOS 必要的安全授权，双击启动。
- [ ] 预期：主窗口在可接受时间内显示，书架和设置可打开，没有隐藏窗口或后台运行但无界面的情况。
- [ ] 失败判定：Dock 显示应用正在运行但长时间无窗口，启动后立即退出，或主页持续空白。

### `APP-02` Windows 全新安装

- [ ] 操作：使用当前 Windows 安装包安装并启动，关闭后再启动一次。
- [ ] 预期：安装、首次启动、二次启动均成功；主窗口可见且能正常退出。
- [ ] 失败判定：安装被阻断、应用闪退、白屏，或关闭后残留进程阻止下次启动。

### `APP-03` 从上一个可用版本覆盖升级

- [ ] 操作：在旧版中准备已导入书籍、阅读进度、书签、高亮、生词和 AI 配置，然后覆盖安装新版。
- [ ] 预期：数据迁移完成，原有书籍可打开，学习数据与阅读位置不丢失，设置页正确展示原作者与当前版本信息。
- [ ] 失败判定：数据库无法打开、书架清空、书籍变成无法读取，或配置被无提示重置。

### `APP-04` 强制退出后恢复

- [ ] 操作：分别在书架、导入中、阅读中和 AI 请求中强制退出，再重新启动。
- [ ] 预期：应用能再次启动，数据不损坏；未完成任务被明确取消、恢复或允许重试。
- [ ] 失败判定：应用无法重启、数据库锁死，或页面永久停留在上次的加载状态。

## 3. 格式导入覆盖矩阵

对每种受支持格式执行相同的基本链路：选择文件 → 观察导入状态 → 确认书架信息 → 首次打开 → 连续阅读三个章节/页面 → 关闭再打开。

| 用例 | 格式 | 必须准备的样本 | 格式特有检查 |
| --- | --- | --- | --- |
| `IMP-EPUB` | EPUB | EPUB 2、EPUB 3、有/无目录、带图片、带脚注 | 目录顺序、图片、内嵌字体、脚注链接 |
| `IMP-PDF` | PDF | 文本型、扫描型、竖版、横版、加密文件 | 页数、缩放、搜索/选词能力边界、密码提示 |
| `IMP-TXT` | TXT | UTF-8、UTF-8 BOM、UTF-16 LE/BE、GB18030、中英文混排 | 编码识别、分章、换行、内部 EPUB 转换 |
| `IMP-MD` | Markdown | 多级标题、表格、代码块、图片、链接 | 标题层级、原始标记不外露、图片与链接 |
| `IMP-HTML` | HTML/HTM | 单文件、本地图片/CSS、不完整 HTML | 字符集、资源路径、脚本安全、样式隔离 |
| `IMP-MOBI` | MOBI | 旧式 MOBI、带封面/目录 | 转换结果、封面、编码、目录 |
| `IMP-AZW` | AZW/AZW3 | 无 DRM 样本、有 DRM 样本 | 无 DRM 可读；对 DRM 限制给出准确提示 |
| `IMP-FB2` | FB2/FBZ | 单书、压缩包、带封面 | XML 编码、嵌入图片、压缩解包 |
| `IMP-CBZ` | CBZ | 规范页码、乱序文件名、大图、混合图片格式 | 页面排序、缩放、方向、内存使用 |

### 每种格式的基本判定

- [ ] 操作：从文件选择器导入一个合法样本，记录从点击确认到书架可用的耗时。
- [ ] 预期：支持的扩展名可选，导入状态可见，书籍只生成一条记录，书名、作者、封面和格式正确。
- [ ] 操作：立即打开书籍，进入第一章/页，再通过目录跳到中间与末尾。
- [ ] 预期：正文显示完整，章节/页数正确，进度随定位变化，内容没有乱码、截断或重复。
- [ ] 操作：关闭阅读页，再次打开同一本书。
- [ ] 预期：能再次打开，并恢复到上次的有效阅读位置。
- [ ] 失败判定：支持的文件不可选、导入无反应、书架记录存在但无法打开、章节数为 0、内容为空，或重启后书籍丢失。

## 4. TXT 导入与内部 EPUB 转换专项

### `TXT-01` 中英文常规小说

- [ ] 操作：导入一本约 200 KB、1000 行、具有 5 个以上章节的 UTF-8 TXT，导入完成后立即打开。
- [ ] 预期：TXT 被转换为结构完整的内部阅读产物；第一章正文显示，章节数不为 0，进度定位事件能触发。
- [ ] 失败判定：页面显示 `Chapter 0 of N`、持续 `Preparing book... 0%`、首章 iframe 未就绪，或没有任何超时错误提示。

### `TXT-02` 编码与 BOM

- [ ] 操作：逐一导入 UTF-8、UTF-8 BOM、UTF-16 LE、UTF-16 BE 和 GB18030 的同内容样本。
- [ ] 预期：可支持的编码均无乱码、无 BOM 可见字符；不支持的编码在导入阶段明确拒绝并说明原因。
- [ ] 失败判定：静默使用错误编码转换、书籍可导入但全文乱码，或在打开阶段才卡死。

### `TXT-03` 分章规则

- [ ] 操作：分别导入使用 `Chapter 1`、`第一章`、罗马数字标题、纯数字标题、无明显章节标题的 TXT。
- [ ] 预期：章节划分可预期；无标题文件仍生成至少一个可读章节，不会生成空 spine。
- [ ] 失败判定：章节内容遗失、同一段重复进入多章、错误分成大量空章，或目录与实际正文不对应。

### `TXT-04` 换行、空白与超长行

- [ ] 操作：导入 CRLF、LF、CR、大量空行、每段只有一行、全书少量超长行的样本。
- [ ] 预期：段落结构可读，不溢出页面，不因超长行阻塞转换或渲染。
- [ ] 失败判定：全文粘成一段、字符溢出、窗口无响应，或内存持续上升。

### `TXT-05` 源格式与渲染产物一致性

- [ ] 操作：导入 TXT，对照书架元数据、数据库记录、内部文件扩展名/MIME 类型与实际交给阅读引擎的类型。
- [ ] 预期：用户可见的源格式可保留为 TXT，但渲染引擎收到的是与内部 EPUB 产物一致的文件名、MIME 类型和字节内容。
- [ ] 失败判定：用 `source_format=txt` 包装实际 EPUB 字节，使引擎按 TXT 路径解析 EPUB；或数据库指向不存在/未完成写入的产物。

## 5. 异常输入与边界文件

### `EDGE-01` 空文件与只含空白的文件

- [ ] 操作：导入 0 字节文件和只含空格/换行的 TXT/Markdown/HTML。
- [ ] 预期：导入被拒绝，提示“文件无可读内容”，不生成书架记录或残留产物。
- [ ] 失败判定：显示导入成功但打开后卡住，或产生无法删除的空书。

### `EDGE-02` 损坏或伪造格式

- [ ] 操作：导入被截断的 EPUB/PDF/CBZ、扩展名与内容不符的文件、缺少必要 manifest/spine 的 EPUB。
- [ ] 预期：在可能的最早阶段检出，显示可理解的错误，不进入无期限阅读准备状态。
- [ ] 失败判定：导入对话消失但无结果、书架出现幽灵记录，或应用崩溃。

### `EDGE-03` 特殊文件名与路径

- [ ] 操作：导入包含中文、英文、阿拉伯文、emoji、空格、多个句点、括号、`#`、`%`、超长文件名的文件，并从多层中文目录导入。
- [ ] 预期：文件可导入、打开、删除和再导入；界面文本不溢出，路径不被错误 URL 解码。
- [ ] 失败判定：找不到文件、书名被截断为空、资源路径 404，或删除操作作用于错误文件。

### `EDGE-04` 大文件和大量书籍

- [ ] 操作：分别导入 10 MB、50 MB、200 MB 的文本/图文书，以及批量导入 10、50、200 本书。
- [ ] 预期：进度持续更新，窗口可交互；到达限制时明确告知用户，不会因内存或磁盘压力导致静默失败。
- [ ] 失败判定：界面长时间无响应、进度不变且无阶段信息、内存不回落、产生半成品，或应用被系统终止。

### `EDGE-05` 磁盘空间不足与权限异常

- [ ] 操作：在书库所在卷空间不足、目录只读、文件被锁定的情况下导入。
- [ ] 预期：写入前或写入失败时明确提示空间/权限问题，数据库与文件系统保持一致。
- [ ] 失败判定：已写入数据库但文件不完整，重试一直命中损坏缓存，或错误被显示为不支持格式。

## 6. 导入过程中断与并发

### `FLOW-01` 取消导入

- [ ] 操作：在文件读取、解析/转换、封面生成、数据库写入阶段分别取消。
- [ ] 预期：取消后任务终止，临时文件被清理，不生成不可用的书籍记录。
- [ ] 失败判定：点击取消仍在后台运行、稍后突然出现书籍，或影响下一次导入。

### `FLOW-02` 导入中关闭窗口/退出应用

- [ ] 操作：在各导入阶段关闭窗口或退出应用，再次启动。
- [ ] 预期：重启后能识别未完成任务，进行清理、恢复或提示重试；不把半成品当作已完成书籍。
- [ ] 失败判定：启动后崩溃、数据库被锁、书籍卡在永久导入中状态。

### `FLOW-03` 重复导入

- [ ] 操作：连续导入同一文件，再导入“同名不同内容”和“不同名相同内容”的文件。
- [ ] 预期：重复判定规则一致，覆盖/保留两本/取消的结果可预期，不误删阅读数据。
- [ ] 失败判定：生成多个无法区分的记录、旧书进度被错误绑定到新书，或发生文件覆盖但元数据未更新。

### `FLOW-04` 并发导入与立即打开

- [ ] 操作：同时导入多本大小不同的书；一本刚显示在书架时立即双击打开。
- [ ] 预期：任务状态相互独立，书籍只在渲染产物已原子性写入后变为可打开。
- [ ] 失败判定：进度串到另一本书、产物路径互相覆盖，或阅读器读取到未完成文件而卡在 0%。

## 7. `Preparing book` 和首屏渲染专项

### `OPEN-01` 加载阶段可见性

- [ ] 操作：首次打开一本需转换的 TXT 和一本原生 EPUB，记录界面从路由进入到显示正文的所有状态。
- [ ] 预期：至少能区分“读取文件”、“解析书籍”、“初始化阅读器”、“加载首章”；状态文字与实际阶段一致。
- [ ] 失败判定：整个过程只显示 `Preparing book... 0%`，无法判断是转换、解析还是渲染卡住。

### `OPEN-02` 各阶段超时与退路

- [ ] 操作：通过损坏资源、不可达路径、缓慢磁盘或大文件分别使文件读取、解析、`view.init()` 和首章加载不能正常完成。
- [ ] 预期：每个可挂起的阶段都有明确的最长等待时间；超时后显示具体错误、重试和返回书架操作。
- [ ] 失败判定：任一 Promise/事件可以永久不解析，且界面无超时、无错误、无取消。

### `OPEN-03` 首章与阅读器定位事件

- [ ] 操作：打开无历史进度的书，记录阅读器初始化、首章 iframe/document 加载、首次 `relocate` 的先后顺序。
- [ ] 预期：首章加载后一定进入可见阅读状态；首次定位事件可正常更新章节、页码/位置与进度。
- [ ] 失败判定：必须等到 `relocate` 才隐藏加载层，但首章失败时永远不触发该事件；或正文已出现但加载层仍遮挡。

### `OPEN-04` 首次、第二次与多窗口打开

- [ ] 操作：首次打开后关闭，连续打开同一本书 10 次；快速双击书籍；从两个窗口打开同一本书。
- [ ] 预期：每次都能进入正文，不重复转换；多窗口策略明确，事件监听与资源能在关闭时清理。
- [ ] 失败判定：偶发 0%、出现多个重叠阅读器、进度互相覆盖，或每次打开内存都持续增长。

### `OPEN-05` 窗口状态变化

- [ ] 操作：加载中最小化、切换应用、锁屏/唤醒、缩放窗口、全屏/退出全屏。
- [ ] 预期：恢复可见后仍能完成渲染，尺寸重算不会把正文移出可见区域。
- [ ] 失败判定：唤醒后空白、进度回到 0%、正文被遮挡，或只有改变窗口大小后才显示。

## 8. 阅读交互与排版

### `READ-01` 翻页、滚动与目录定位

- [ ] 操作：使用按钮、键盘、触摸板连续翻页；切换滚动/分页模式；从目录跳转至首、中、末章。
- [ ] 预期：顺序正确，无空白页、重页或跳页；目录高亮和进度同步。
- [ ] 失败判定：翻页后卡住、跳到错误章节，或页面显示变化但进度不更新。

### `READ-02` 字体、字号、行高、页边距与主题

- [ ] 操作：逐一修改排版选项，切换浅色/深色主题，关闭再打开书籍。
- [ ] 预期：设置立即生效并持久化；文字不裁切、不重叠，当前位置在重排后仍接近原文段。
- [ ] 失败判定：排版变更导致空白、跳回章首、选词坐标错位，或主题导致文字与背景对比不足。

### `READ-03` 不同窗口尺寸与系统缩放

- [ ] 操作：在最小窗口、常规窗口、全屏、125%/150% Windows 缩放和 macOS Retina 屏幕下阅读。
- [ ] 预：工具栏、正文、查词弹窗和侧边栏不重叠或溢出，关键按钮始终可达。
- [ ] 失败判定：控件挡住正文、弹窗移出窗口，或动态内容使布局持续跳动。

## 9. 阅读进度、书签、高亮与生词

### `STATE-01` 进度保存与恢复

- [ ] 操作：在章首、章中和末章分别退出阅读页，重新打开；退出整个应用后再打开。
- [ ] 预期：恢复到上次的语义位置，书架进度与阅读页一致，不因排版改变恢复到无效位置。
- [ ] 失败判定：每次打开回到 0%、跳到别的章节，或定位失败时页面卡住而不回退到章首。

### `STATE-02` 书签和高亮生命周期

- [ ] 操作：创建、编辑、删除书签与多色高亮，从列表跳回原文，关闭应用后重试。
- [ ] 预期：标注与准确文本/位置绑定，重启后保留，删除后不再出现。
- [ ] 失败判定：文字重排后高亮偏移、高亮阻断选词，或重新导入其他书后标注串书。

### `STATE-03` 查词与生词收藏

- [ ] 操作：选择单词、词组、跨行文本、带标点文本和中英文混合文本，查询后收藏/取消收藏。
- [ ] 预期：查词框靠近选区且不遮挡必要内容，词条、上下文、书籍与位置保存正确。
- [ ] 失败判定：查词框移出屏幕、查询状态永不结束、收藏丢失，或高亮错误覆盖其他文字。

## 10. AI 释义、侧边栏与对话

### `AI-01` 单词释义与上下文

- [ ] 操作：在不同章节对多义词发起释义，对照发送的选中文本、所在句子、段落和书籍信息。
- [ ] 预期：回答使用正确上下文，界面流式更新且可取消，失败不会阻塞阅读。
- [ ] 失败判定：发送了其他章节内容、一直显示加载、流式输出串到上一次请求，或密钥/完整请求泄露到用户界面。

### `AI-02` 多 API 密钥切换

- [ ] 操作：配置两个有效密钥；再分别模拟第一个密钥的 401/403、429、5xx、超时和网络中断。
- [ ] 预期：只对指定的可切换错误自动使用下一密钥；请求有总重试上限与冷却时间，不重复计费，不在日志中暴露密钥。
- [ ] 失败判定：任何错误都盲目切换、两个密钥无限循环、已输出部分回答后重发造成重复，或所有密钥不可用时无明确错误。

### `AI-03` 可展开侧边栏与追问

- [ ] 操作：从查词结果打开侧边栏，连续追问；折叠/展开、切换章节、切换书籍再返回。
- [ ] 预期：侧边栏保留当前书籍与词条的对话上下文，不遮挡全部正文；新书的上下文不与旧书混用。
- [ ] 失败判定：追问丢失原释义、折叠后流式任务中断且无状态、侧边栏使正文宽度为 0，或对话串书。

### `AI-04` 断网、恢复与并发操作

- [ ] 操作：请求前断网、流式回答中断网、恢复网络；快速发起多次查词并在等待时翻页。
- [ ] 预期：网络错误和 API 配置错误被区分，可取消/重试；旧请求不覆盖新选词的结果。
- [ ] 失败判定：重试无上限、旧结果出现在新词条下，或 AI 失败导致阅读器无法翻页。

## 11. iCloud 和本地文件可达性

### `SYNC-01` iCloud 正常可用

- [ ] 操作：选择 iCloud 书库，导入书籍，创建进度、高亮和生词，退出后再启动。
- [ ] 预期：书籍与数据均持久化，界面能区分“已保存到本地”与“已同步到 iCloud”。
- [ ] 失败判定：仅创建了目录便显示同步成功、状态长期虚假成功，或重启后书籍不可达。

### `SYNC-02` iCloud 目录暂时不可达

- [ ] 操作：在 iCloud 未登录、同步暂停、目录未挂载和网络断开时新增/修改阅读数据，然后恢复 iCloud。
- [ ] 预期：变更先可靠保存到本地队列；目录恢复后自动发布，成功前不丢弃队列项。
- [ ] 失败判定：因目录不可达而不进入离线队列、本地显示已保存但重启后丢失，或恢复后需用户重做所有修改。

### `SYNC-03` iCloud 占位文件与恢复下载

- [ ] 操作：将书籍文件设为仅云端占位，断网打开；恢复网络并等待下载后重试。
- [ ] 预期：离线时明确说明文件尚未下载，不把占位文件当成损坏书籍；下载完成后可重试打开。
- [ ] 失败判定：页面永久 0%、错误地删除书籍记录，或实际文件下载后仍使用之前的失败缓存。

### `SYNC-04` 冲突、原子写入与孤立文件

- [ ] 操作：在两个设备修改同一本书的进度/高亮；在同步写入中断网；手动删除书籍文件或数据库记录之一。
- [ ] 预期：冲突规则可预期并尽可能保留非冲突数据；写入使用临时文件和原子替换；孤立状态可检测且可恢复/清理。
- [ ] 失败判定：后同步设备静默覆盖较新数据、同步到半个 JSON/EPUB，或孤立文件使同步队列永久失败。

## 12. 删除、重新导入与数据一致性

### `DATA-01` 删除书籍

- [ ] 操作：对无学习数据、有进度/高亮/对话的书分别执行删除，取消一次确认，再真正删除。
- [ ] 预期：确认信息说明影响范围；取消无任何变化；确认后按产品策略完整删除或保留相关数据。
- [ ] 失败判定：只删数据库不删文件、只删文件不删记录，或删除一本书后其他书的高亮/对话丢失。

### `DATA-02` 删除后重新导入

- [ ] 操作：删除一本书后立即重新导入原文件，再导入修订过的同名版本。
- [ ] 预期：新书使用新产物与元数据，是否恢复旧学习数据符合明确的产品规则。
- [ ] 失败判定：命中已删除书的损坏缓存、显示旧封面/章节，或新旧书 ID 错误共用数据。

### `DATA-03` 数据库与文件系统对账

- [ ] 操作：对导入成功、导入失败、取消、删除、同步冲突后的数据库记录、书籍产物、封面、临时文件和队列记录进行对账。
- [ ] 预期：每条可用书籍记录都有完整产物；无记录指向临时文件；无无主的大文件或永久 pending 任务。
- [ ] 失败判定：任意孤立记录、孤立产物、未清理临时文件，或多本书指向同一可变产物。

## 13. 性能、资源与稳定性

### `PERF-01` 导入和首次打开耗时

- [ ] 操作：对小/中/大型各格式样本各执行 5 次导入和首次打开，记录中位数、最大值和失败次数。
- [ ] 预期：指标满足团队预先确定的发布预算；界面进度更新不长时间停滞。
- [ ] 失败判定：没有性能预算、同一样本耗时波动巨大、第二次打开仍重复转换，或出现任意无限等待。

### `PERF-02` CPU、内存与磁盘

- [ ] 操作：记录空闲、导入、阅读、快速翻页、AI 流式回答和关闭书籍后的 CPU、内存和磁盘占用。
- [ ] 预期：任务结束后 CPU 回到空闲水平，关闭阅读页后大部分书籍资源释放，临时文件可清理。
- [ ] 失败判定：卡在 0% 时 CPU 长时间满载、重复打开后内存单调增长，或每次转换留下大量临时文件。

### `PERF-03` 长时间阅读

- [ ] 操作：持续阅读 2 小时，期间反复翻页、查词、高亮、打开/关闭侧边栏并使应用多次进入后台。
- [ ] 预期：操作响应不明显恶化，进度持续保存，无重复事件和资源泄漏。
- [ ] 失败判定：翻页越来越慢、同一操作触发多次、内存持续上升，或后台恢复后无法继续阅读。

## 14. 日志、错误提示与可恢复性

### `OBS-01` 链路日志

- [ ] 操作：各执行一次导入成功、转换失败、阅读器初始化超时、首章失败和 iCloud 不可达。
- [ ] 预期：日志可用一个任务/书籍关联 ID 串起“选择文件 → 识别格式 → 转换 → 写入 → 打开 → `view.init()` → 首章 → `relocate`”，并包含阶段耗时和错误链。
- [ ] 失败判定：只有“打开失败”而无阶段、书籍或底层原因；日志包含 API 密钥、完整书籍正文或不必要的个人路径。

### `OBS-02` 用户可见错误

- [ ] 操作：触发不支持格式、编码错误、损坏文件、读取权限、磁盘满、渲染超时、iCloud 占位文件和 AI 错误。
- [ ] 预期：错误文案区分原因，用普通用户能理解的语言说明下一步，同时提供可用于支持排查的错误编号。
- [ ] 失败判定：只显示技术堆栈、所有情况都显示“未知错误”、提示一闪而过，或错误层阻止用户返回。

### `OBS-03` 重试、取消与降级

- [ ] 操作：在各类可恢复错误上分别执行重试、取消和返回，连续重试 3 次。
- [ ] 预期：重试会重建必要状态，不复用已 reject 的 Promise 或损坏引擎实例；取消会停止后台任务；返回书架后可打开其他书。
- [ ] 失败判定：点击重试没有新请求、重复创建多个阅读引擎，或取消后迟到回调仍覆盖当前页面。

## 16. 问题分级与发布退出标准

### 问题分级

| 级别 | 定义 | 本清单典型例子 | 发布处置 |
| --- | --- | --- | --- |
| P0 | 大面积数据丢失、安全事故、应用普遍无法启动 | 升级后书库清空；API 密钥泄露 | 立即停止发布 |
| P1 | 核心流程无法完成且无可接受规避方法 | 受支持 TXT 导入后永久 0%；主窗口不显示 | 阻止发布 |
| P2 | 重要功能错误，但有明确规避方法 | 特定编码乱码；高亮位置偏移 | 原则上修复；例外放行需记录风险 |
| P3 | 轻微视觉、文案或低概率体验问题 | 非关键元数据截断、提示文案不够准确 | 可带已知问题发布 |

### 发布退出标准

- [ ] P0 和 P1 问题数量均为 0。
- [ ] 所有声明支持的格式均至少有一个正常样本和一个异常样本通过。
- [ ] TXT 的编码、转换产物、首章渲染、进度事件和重启恢复用例全部通过。
- [ ] 没有任何导入或打开路径可长期停留在加载状态而无超时、错误和退路。
- [ ] 失败、取消和强制退出后，数据库、阅读产物和临时文件之间无不一致。
- [ ] macOS Apple Silicon 和 Windows 的实际发布包均完成安装、启动、导入、阅读、进度恢复的核心回归。
- [ ] 每个未关闭的 P2/P3 问题都有负责人、影响范围、规避方法和计划修复版本。

- [ ] `SMOKE-01`：全新启动，主窗口可见。
- [ ] `SMOKE-02`：导入 UTF-8 TXT，书架信息正确。
- [ ] `SMOKE-03`：导入后立即打开 TXT，首章正文显示，不停在 0%。
- [ ] `SMOKE-04`：TXT 首次打开时渲染引擎收到 EPUB 产物对应的类型。

## 18. 问题报告最小内容

任一检查失败时，问题报告至少应包含：

- [ ] 完整操作步骤，包括每步等待了多久。
- [ ] 预期结果与实际结果。
- [ ] 第一次异常出现的准确阶段，不仅是最终页面状态。
- [ ] 应用版本、平台、存储方式、网络和 iCloud 状态。
- [ ] 可安全分享的最小复现文件，并记录文件大小、编码、章节数和哈希。
- [ ] 从导入开始到失败后返回/退出的录屏或带时间戳截图。
- [ ] 与该书籍/任务关联的完整日志片段，但已脱敏 API 密钥、用户路径和书籍正文。
- [ ] 是否可稳定复现、复现次数/尝试次数，以及重启、重新导入或换存储位置是否可规避。
- [ ] 失败后的数据一致性：数据库记录、渲染产物、临时文件和离线队列是否残留。

# Lantern

[简体中文](README.md) · [English](README.en.md)

> 不必适应一套固定的 AI 阅读方式。
>
> Lantern 让 AI 用你能理解的语言解释书，也让你按自己的目标重塑阅读工具。

Lantern 是一款面向英语学习与长篇阅读的桌面应用，以 macOS 为主要平台。它不只是把“问 AI”放进阅读器：你可以决定 AI 如何解释、解释到什么深度、卡片展示哪些信息，以及选中文本后该触发什么操作。

从轻量双击查词，到专注长难句、文学分析、考试精读或专业阅读的 AI 工作流，Lantern 都可以逐步调成适合你自己的样子。

这是基于开源项目 [Quill](https://github.com/yicheng47/quill) 独立维护的个人版本，并非原项目的官方发行版。

## 英语解释，应当与你的英语水平匹配

许多阅读工具虽然提供“英语解释”，却往往直接使用接近母语者的词汇和复杂句式：一个不认识的词，最后换来一段更难读懂的英文。

Lantern 将你的英语水平作为解释条件，而不只是一个界面选项。你可以在个人资料中填写当前的 CEFR 等级（A1–C2），也可记录 IELTS、TOEFL、TOEIC、剑桥英语、DET 或四六级等成绩，用于估算学习等级。

每次查词、短语释义和段落解读时，AI 都会同时参考你的等级、解释语言、目标译文和内容密度：

- **A1–A2**：以准确中文义为基础，并配合符合当前水平的简明英文，避免“用更难的英文解释简单英文”。
- **B1**：以英文解释为主；对抽象、容易误解的部分保留必要的中文辅助。
- **B2–C2**：提供更自然、更深入的英语解释，并可按需展示目标语言翻译。
- 选择“英语优先”并不等于接受难懂的母语级解释；AI 会主动控制词汇与句子复杂度，使解释本身也成为可理解的输入。

你可以从“先看懂”开始，逐渐过渡到“用英语理解英语”，而不必被一段看不懂的英文解释再次打断阅读。

## 预设模块只是起点：不满意，就自己打造

Lantern 提供语境含义、词性、常见义项、搭配、语法分析、指代关系、语气、写作表达、记忆提示等预设学习模块；它们都可以按单词、短语、句子/段落分别配置开关、顺序、默认展开状态和内容密度。

但预设不应限制你的学习方法。

如果现有模块不能满足需求，你可以创建自己的 AI 模块：

- 为模块命名，并编写完全属于自己的提示词。
- 让它基于当前选区、上下文和书籍信息生成内容。
- 将它加入单词、短语或段落学习卡片，与内置模块一起显示、隐藏、排序和设置展开状态。
- 测试、优化提示词，直到输出符合你的习惯。
- 创建自定义选区操作，并绑定快捷键或双击操作，一键执行自己的阅读方法。

例如，你可以制作：

- 面向雅思或考研的“长难句拆解”模块；
- 用于文学作品的“修辞、叙事视角与语气”模块；
- 用于专业书籍的“术语、前置知识与应用场景”模块；
- 用于写作积累的“可复用表达与改写建议”模块；
- 只保留最少信息、避免打断沉浸阅读的极简查词模块。

## 你决定 AI 告诉你什么、何时告诉你

- 对单词、短语、句子和段落分别设计学习卡片。
- 自由显示、隐藏、排序内置或自定义模块。
- 在“简洁、标准、详细”间控制信息密度，并设置例句与关键术语数量。
- 配置查词语言、解释语言、翻译目标语言，以及是否显示简短译文。
- 自定义选区菜单的操作和排列：查词、释义、翻译、收藏、标记、复制、问 AI 或自定义操作。
- 单击打开菜单、双击快速查词，也可为选区操作设置快捷键。
- 在设置中即时预览卡片；确认需要时，再调用真实 AI 服务查看结果。

## AI 服务由你选择

- 支持 OpenAI 兼容 API、Anthropic、Ollama 和可选的 OpenAI OAuth。
- 可添加多个 AI 服务，分别配置名称、Base URL、模型和优先级。
- 支持自定义兼容 API，例如自建网关或其他 OpenAI 兼容服务商。
- 每个服务可保存多个 API Key；在开始输出前，会按优先级自动尝试可用 Key 和服务。
- 支持连接测试和模型发现。
- API Key 与 OAuth 令牌仅保存在当前设备的本地凭据数据库中，不会返回给 Webview，也不会参与同步。

## 从阅读到学习闭环

- 通过 AI 查词、短语释义、段落解读和翻译理解当前内容。
- 在同一本书内持续对话，将单词、原句、释义和上下文带入 AI 面板继续追问。
- 保存词汇、查询历史、学习状态和笔记，并能回到原文位置。
- 使用“新词、学习中、已掌握”等状态管理词汇，并复习到期内容。
- 导出或导入 CSV/JSON 词汇备份。
- 查词后自动标记单词；可选择只标记当前位置，或标记全书相同单词。
- 自定义手动和自动标记的颜色、透明度、高亮/下划线/粗体样式及字体。

## 阅读体验也可定制

- 导入自定义字体，调整字体、行距、页边距和阅读布局。
- 选择主题或自定义页面背景色与色调。
- 在滚动和分页阅读之间切换；PDF 支持缩放、单页/双页与滚动/分页布局。
- 支持书签、高亮、笔记、目录和阅读进度。
- 可在不改变原书内容的前提下，为学习痕迹添加轻量标记。

## 系统要求

| 平台 | 支持范围 |
| --- | --- |
| macOS | **macOS 12 Monterey 或更高版本**，仅支持 **Apple Silicon（M 系列）** Mac |
| Windows | 提供 **Windows 11 x64** 安装包 |
| Intel Mac | 当前不提供 Intel macOS 安装包 |
| Linux | 当前未提供发布版本 |

macOS 是主要平台。基于用户选择的 iCloud Drive 文件夹的多设备同步仅在 macOS 上可用；Windows 版本可本地阅读和使用 AI 功能，但不包含该 iCloud 同步能力。

## 文件格式能力

| 源格式 | 导入方式 | 阅读控制 | 选择与手动高亮 | 自动词汇标记 |
| --- | --- | --- | --- | --- |
| EPUB | 原生阅读 | 字体、行距、页边距、滚动/分页流式阅读 | 支持 | 支持 |
| TXT、Markdown、HTML | 保留原始文件，并转换为稳定的内部 EPUB | 与 EPUB 相同 | 支持 | 支持 |
| PDF | 原生阅读 | 主题、缩放、单页/双页布局、滚动/分页阅读 | PDF 具备可用文本层时支持 | 首个版本未提供 |
| MOBI、AZW、AZW3、FB2、FBZ | 通过 Foliate 原生解析器阅读 | 渲染器支持时可使用流式阅读控制 | 暂未开放 | 不支持 |
| CBZ | 原生阅读 | 仅主题 | 不支持 | 不支持 |

文件格式支持描述的是当前本地导入和阅读器集成能力，并不代表支持 DRM，也不保证能完美渲染每一种出版商特定的文件变体。

## 本地优先与同步

书库数据以本地为先。若需要多设备同步，请在“设置”中选择自己 iCloud Drive 内的一个文件夹，并在每台 Mac 上选择同一个文件夹。应用会将事件日志、书籍和封面保存在那里。

当前版本不使用原版 Quill 的 iCloud 容器，也不宣称与原版 Quill 的 iOS 应用或其私有 iCloud 数据兼容。

## 开发

要求：Node.js 22、npm、Rust，以及目标平台所需的 Tauri 前置依赖。阅读器引擎源码已随仓库一并提交：

```bash
git clone https://github.com/KlaraGraff/lantern.git
cd lantern
npm ci
npm run tauri dev
```

常用静态检查：

```bash
npm exec tsc --noEmit
npm run lint
cd src-tauri && cargo check
```

仓库协作约定见 [AGENTS.md](AGENTS.md)。
